Get paid for pull requests

To encourage transparency and improve the performance and functionality of our website, Binary.com has open-sourced the entire front-end of our codebase, and we are now actively encouraging the community to contribute to its development.

We'll pay for any pull request that is accepted and merged into the master codebase.

Which projects are available?

Seluruh platform dan perpustakaan Binary.com adalah memenuhi syarat.

How do I submit a pull request?

  • Fork the relevant Github repository
  • Make your code changes in your fork
  • Submit a pull request to master in the original GitHub repository
  • Send an email to pull-requests@binary.com to ask our development team to review your pull request

How do I test my changes?

Semua proyek front-end Binary.com dikerahkan dengan menggunakan fasilitas halaman Github. Biasakan diri Anda menggunakan fasilitas tersebut, dan baca petunjuk penerapan pada setiap repositori terkait.

What kind of changes can I submit?

  • New features
  • Fasilitas charting yang lebih baik
  • New technical indicators
  • Improved usability
  • Improved browser compatibility
  • Improved responsiveness
  • Faster rendering
  • Perbaikan bug
  • Improved documentation
  • Cakupan tes yang lebih baik

Dapatkah saya menulis front-end baru dari awal?

Sure. We welcome projects such as new mobile apps, chatbot apps, browser extensions, and new charting tools.

Dapatkah saya mengirimkan kerentanan keamanan?

Yes, but we'd prefer you to submit security vulnerabilities via our bug bounty program.

Dapatkah saya berkontribusi pada kode back-end?

Our back-end code is mainly in Perl, and our open-source contributions may be found on CPAN. The corresponding source code can be found in our Github repositories that start with the prefix "perl-". We welcome contributions to these repositories, notably improved documentation and test coverage.

How will I get paid?

We only pay for pull requests that are approved by our development team and merged into master. If the development team approves your pull request, we'll pay you in cryptocurrency. You can choose Bitcoin, Litecoin or Ether.

What can I expect to get paid?

Our development team will determine how much you get paid. You can expect the following range of payments:

  • Trivial changes – generally $5-20
  • Smallish changes – generally $20-50
  • Perubahan ukuran rata-rata – umumnya $50-250
  • Medium-sized changes – generally $250-750
  • Large changes – generally $750-2,500
  • Very large changes/entire new projects – $2,500-10,000

Apakah saya perlu mempertahankan kepemilikan kode?

No. Once your pull request is merged into master by our development team, and you have been paid, you have effectively handed over the rights to the code. It then becomes part of the master Binary.com codebase under the open-source license of the relevant repository.

If rewards are not agreed in advance, how do I know I'll be paid fairly?

We want to keep improving our platform, so, it's in our interest to pay contributors fairly. If we don't, then you'll stop contributing pull requests, which will make this project self-defeating.

Dapatkah saya bekerja untuk Binary.com penuh-waktu?

We are always on the lookout for talent – please apply via our careers page.

Dapatkah saya membahas codebase dengan tim pengembangan Binary.com?

Of course! Please send an email to pull-requests@binary.com to request an invite to our Slack channel.
